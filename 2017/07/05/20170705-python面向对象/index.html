面向对象

  • 在 Python 中，万物皆对象，任何一个事物实例都可以比作一个对象，都拥有：属性与行为。
  • 面向过程与面向对象都是解决问题的思路。

类

类就是对某一个具体对象的抽象。如：奥拓、奥迪、奔驰等可以抽象成一个汽车类，而它们就是这个汽车类的实例对象。

定义类，类名遵守驼峰规则，首字母大写

使用类创建对象

  • 属性 name,color 都是汽车类的属性，属性分为类属性与对象属性。
  • 方法 run 是汽车类的方法，方法分为类方法与对象方法。
  • init 方法是构造方法，用于创建差异性的实例对象。

抽象类与抽象方法

抽象类： 一个抽象类并不是一个具体化的类，不能直接创建实例的类。
抽象方法： 抽象出来的一个方法，没有函数体，不能直接调用。

abc 模块

python 中不直接支持抽象类与抽象方法，需要借助 abc 模块。

  • metaclass=abc.ABCMeta 声明为抽象类
  • @abc.abstractmethod 声明为抽象方法

类的SOLID设计原则

单一职责原则

一个类只负责一项职责

好处：

  • 易于维护，写出高内聚的代码
  • 易于代码复用

案例：

计算器类，如果设计的时候包含加、减、乘、除和结果打印。那如果后期需要添加结果写入文件，结果上传服务器等操作，就需要再次去修改计算器类，不利于后期维护与代码复用。所以，我们设计计算器类，应该只包含加、减、乘、除方法，而结果相关的应该分离到别的合适的类中去。

开放关闭原则

对扩展开发，对修改关闭

好处：

  • 易于维护，保证代码安全性与扩展性

案例：

在 Person 类如果定义了一个让宠物工作的方法，那么每次新增一个宠物，可能都会要修改到这个工作方法。这时候，就适合使用抽象类抽象方法解决。

里氏替换原则

使用基类引用的地方必须能使用继承类的对象

好处

  • 防止代码出现不可预知的错误
  • 方便针对基类的测试代码，可以复用在子类上

案例：

小鸟类，定义了一个 飞 的方法。但是如果是鸵鸟，鸵鸟不会飞，所以不可以使用鸵鸟替换小鸟。

接口分离原则

如果一个类包含了过多的接口方法，而在这些方法使用的过程中并非不可分割，那么应当把他们进行分离。在python中，抽象方法可以当做为接口。

好处

  • 提高接口的重用价值

案例

小鸟：吃，叫，飞。但如果是鸵鸟，那么飞就不是必须的，就需要把飞分离出去。

依赖倒置原则

高层模块不应该直接依赖低层模块，应该依赖抽象类或者接口。

好处

  • 利于代码维护

案例

电脑类，依赖的不是某一个具体的鼠标类，而应该是鼠标类的抽象。能单机、双击、右击、移动等功能。具体的实现类可能是触摸板、有线鼠标、无线鼠标。
